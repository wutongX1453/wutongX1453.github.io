发表于 2025-09-14

Godot概论

Godot 是一个免费、开源的 2D 和 3D 游戏引擎。它提供了一个完整的、集成的开发环境，让开发者可以轻松地创建游戏和应用程序。

你可以把它理解为类似于 Unity 或 Unreal Engine 的替代品，但它以其独特的设计哲学和完全开源的特性而闻名。

核心特点与优势

  1. 完全免费和开源（MIT 许可证）这是 Godot 最大的优势之一。你可以完全免费地使用它，无论是个人学习、业余爱好还是商业项目，都无需支付任何费用或版税。它的源代码公开，意味着社区可以不断改进它，透明度极高。
  2. 轻量级与高性能整个引擎就是一个可执行文件，非常小巧（几十MB），下载和启动速度极快，对电脑硬件要求较低。
  3. 场景与节点架构这是 Godot 设计理念的核心。游戏中的一切（角色、敌人、灯光、UI元素等）都是一个节点（Node）。将这些节点像搭积木一样组合起来，就形成了场景（Scene）。这种结构非常直观灵活，易于理解和复用。
  4. 内置脚本语言 GDScriptGodot 自带一门叫做 GDScript 的脚本语言。它的语法类似于 Python，非常简洁易学，专为游戏开发和高迭代效率而设计，与引擎的集成度极高。同时，它也官方支持其他语言，如 C# 和 VisualScript（节点式编程），并且通过社区扩展支持更多语言（如 Rust）。
  5. 强大的 2D 支持Godot 的 2D 引擎是其强项。它专为 2D 开发设计，拥有高效的像素精确渲染和直观的 2D 工作流程，许多开发者认为其 2D 体验甚至优于一些主流商业引擎。
  6. 活跃且友好的社区作为一个开源项目，Godot 拥有一个非常活跃、乐于助人的全球社区。当你遇到问题时，很容易在论坛、Discord 或视频网站上找到帮助和教程。
  7. “一应俱全”的设计引擎内置了大量工具，几乎涵盖了游戏开发的所有方面，包括动画、粒子、物理、导航、UI 编辑等，减少了对外部工具的依赖。

nullptr

  • NULL：在 C 中，NULL通常是一个宏定义，可能是 (void*)0或简单的 0。在 C++ 中，NULL通常定义为 0或 0L（整数类型）。
  • nullptr：是 C++11 引入的关键字，具有 std::nullptr_t类型，可以隐式转换为任何指针类型，但不能转换为整数类型。

由于 NULL可能是 0，在函数重载时可能导致歧义

c中使用指针容易出现的错误

  • 悬空指针

指针A由指针B赋值得到，当指针B释放了内存之后，指针A无从得知内存已经被释放

  • 指针越界

  • 内存泄漏

使用完指针后没有及时释放内存，导致可用内存减少

  • 野指针

引用类型

引用与指针不同，他是对象的别名。换句话说引用和被引用对象是同一个对象。

  • 左值引用(&)(长期租客)
  • 右值引用(&&)(临时工住所）

右值引用（&&）是 C++11 引入的重要特性，它使得 移动语义（Move Semantics） 和 完美转发（Perfect Forwarding） 成为可能，极大地提升了 C++ 的性能和表达能力。

  1. 主要规则：不能直接用左值初始化右值引用，必须使用 std::move()进行显式转换。这是一种安全机制，要求程序员明确表示“我同意放弃这个对象的所有权”。
  2. std::move()的本质：它是一个静态转换，不产生任何运行时成本。它只是“允许移动”的信号，而非“执行移动”。

类型自动推导

命名空间

想象你和同事都写了一个 print()函数，但功能不同。如果没有命名空间，编译器无法区分这两个同名函数。命名空间就像给代码加了一个“姓氏”，避免重名混淆。

  • Define
  • Use

:: 是作用域解析运算符，指明调用哪个命名空间中的函数。

  • auto(弃用)
  • static（静动态，主要影响装载）
  • register（建议编译器将变量放入寄存器中）
  • extern
  • mutable(?)
  • thread_local

Lambda匿名函数

引用对象

  • 变量名称是变量附属在内存位置中的标签，您可以把引用当成是变量附属在内存位置中的第二个标签。因此，您可以通过原始变量名称或引用来访问变量的内容。
  • 对r的操作实际上就是对i的操作，r是i的别名
  • 在简单场景下直接用变量就行，引用主要用于需要“别名”或高效传递的场合，特别是函数参数和返回值。简化指针操作,引用比指针语法更直观、安全。

stdio与iostream

  • stdio是 C 语言的 I/O 库，基于函数和格式字符串。
  • iostream是 C++ 的 I/O 库，基于类、对象、流和运算符重载。
  • iostream底层实现通常（但不必须）会利用 stdio的设施（如 FILE*和缓冲区）。
  • iostream解决了 stdio在类型安全和可扩展性方面的主要缺点。

C++ 程序中可以同时使用两者： C++ 标准库包含了 <cstdio>，因此 C++ 程序可以合法地使用 printf, scanf等函数，也可以使用 cout, cin等流对象。stdio和 iostream通常使用独立的缓冲区。这意味着如果你在同一个程序中混合使用 printf和 cout（或 scanf和 cin），输出的顺序可能不是你代码中调用的顺序，因为数据可能还在各自的缓冲区中未刷新。为了避免混乱的输出顺序，通常需要手动刷新缓冲区

  • 输入examp
  • C++ 编译器根据要输出变量的数据类型，选择合适的流插入运算符来显示值。<< 运算符被重载来输出内置类（整型、浮点型、double 型、字符串和指针）的数据项。
  • 体现了面向对象的设计理念，而stdio中则需要手动在函数中指定输入输出数据类型

结构体与类的区别

在 C++ 中，struct 和 class 本质上非常相似，唯一的区别在于默认的访问权限：

  • struct 默认的成员和继承是 public。

  • class 默认的成员和继承是 private。

基于githubpages与nexo的个人网站搭建

发表于 2025-08-23 更新于 2025-09-15

前言

本教程记录我搭建个人网页的过程，以及步骤原理。前几天电脑被一个破光驱干穿了，更换了新电脑，移植硬盘时不小心把之前Ngnix部署的博客删掉了（放在了很奇怪的路径）。痛定思痛愈发觉得应该选择一个云同步性高的方案,把之前的文章慢慢迁移过来，加上我只是拿来做做笔记什么的静态网页，nexo加githubpages的方案太适合我这种几乎不碰前后端的学生了。

搭建原理

利用nexo(一个静态网页生成器)将你的博客（markdown文档）转换成HTML网页文件，然后利用Git工具上传到一个名为user.github.io的仓库中，这个特殊仓库享受了GitHub免费提供的GitHub Pages静态文件托管服务。GitHub Pages 接受并托管你上传的静态文件，并提供一个特定域名以供访问。收到访问请求后，GitHub Pages直接返回托管的静态网络文件，快速可靠且高效。

准备工作

  • 一个github账号

  • nodejs

    [Windows下载链接](Node.js — Run JavaScript Everywhere)

    Hexo 并非一个开箱即用、双击就能运行的 .exe程序。它是由成千上万行 JavaScript 代码编写而成的。而 Node.js 是一个 JavaScript 运行环境，它允许 JavaScript 脱离浏览器，在操作系统上运行。可以把Hexo看作一个JavaScript工具包，指导Node.js工具进行初始化，部署，预览等工作。

    更关键的是 Node.js提供了一个强大的包管理工具NPM (Node Package Manager)，可以用来下载各种模块化hexo插件。（其实hexo本身也是通过npm安装的）

  • git

    [Windows-X86-64下载链接](Git - Downloading Package)

    hexo 的提交命令依赖Git工具，需要将生成的静态网页（public文件夹）推送到github pages。而且hexo的主题与插件通常由Git分发（我通过git clone 拉取了next主题）。

  • 一颗与网络连接搏斗的心

    教程中大部分问题都是网络连接问题，有时需要反复打开或者关闭科学上网工具，就像USB-A接口总是要插三回一样！
